How it all connects.

Six concepts. One pipeline. Hover any node to see what it does and how it links to the next.

Each concept is one stop in the pipeline that turns raw numbers into intelligent-looking behavior. Together they form a closed loop: parameters compute activations, activations form embeddings, embeddings get searched and ranked, ranks become probabilities, probabilities become the next number predicted.
II · Numbers

Everything starts as a number.

Before there are weights, layers, or embeddings — there are floating-point numbers. The format you choose decides how much memory the model takes, how fast it runs, and how precise it can be.

IEEE 754 — under the hood

A floating-point number is three pieces of information packed together: sign · exponent · mantissa.

A model with 70 billion parameters in FP32 needs 280 GB of memory just to hold its weights. In FP8, that drops to 70 GB. This is why every modern model is shipped in a smaller format than it was trained in.

III · Parameters

A neuron is a weighted sum.

Every parameter in a neural network is either a weight — how much an input matters — or a bias — a baseline value added in. That's it. Billions of them, but always one of those two things.

That's one neuron. A modern frontier model has on the order of 10¹¹ of these, organized into layers. Training is the process of finding the values of all those weights and biases — by gradient descent — so that the network's outputs match what we want.

IV · Embeddings

Words become coordinates.

An embedding turns something messy — a word, an image, a sentence — into a list of numbers. A vector. And vectors live in space, with directions and distances. Suddenly we can do geometry on meaning.

The semantic plane

Real embeddings have hundreds or thousands of dimensions. Here are 30 words flattened to two. Notice that semantic neighbors cluster together.

03SPACE

The fact that "king" is close to "queen" — and that the direction from "man" to "woman" is similar to the direction from "king" to "queen" — is the foundation of every modern AI system that "understands" text. It's also exactly how vector search works.

Cosine similarity = (a · b) / (|a| × |b|). It's the cosine of the angle between two vectors — 1.0 if they point the same way, 0 if perpendicular, -1 if opposite. Production systems do this same calculation against millions of vectors using HNSW or IVF indexes that prune the search space.

VI · Quantization

Smaller numbers, most of the smarts.

Models are trained in high precision but they don't need that precision to run. Quantization replaces each weight with a smaller, coarser approximation. The model gets smaller and faster; the quality, mostly, holds.

Going from FP16 → INT4 cuts memory by 4× with surprisingly little measurable quality loss. Below that, you trade off real capability. The frontier of this work is mixed-precision quantization, where the "important" weights (a small fraction) keep more bits and everyone else gets squeezed.

VII · Output

From numbers back to language.

The model's final layer doesn't output a word. It outputs a logit — a raw score — for every possible next token. Softmax turns those scores into probabilities. Temperature changes how confidently it picks.

Temperature 0 always picks the top token (deterministic). Higher temperatures flatten the distribution — every token becomes more likely — which means more variety and more mistakes. The model itself doesn't change. Just how we read its output.

VIII · Quick Reference

The vocabulary.

Everything we've touched, in a sentence each.

Tensor
A multi-dimensional array of numbers. The data structure that everything in a neural network is. A scalar is 0D, a vector 1D, a matrix 2D, a tensor anything beyond.
Weight
A learned number that scales how much one input contributes to one output. Most parameters in a model are weights.
Bias
A learned constant added after the weighted sum. Lets a neuron output something other than zero when all inputs are zero.
Parameter
A weight or a bias. The numbers the model learns during training and uses during inference. "70B parameters" means 70 billion of these.
Activation
The output of a layer for a specific input — the intermediate values flowing through the model right now. Unlike parameters, these change with every input.
Gradient
The direction of steepest increase of the loss with respect to a parameter. Training nudges each parameter in the opposite direction.
Floating point
A way to store real numbers in a fixed number of bits using sign, exponent, and mantissa. Modern AI runs on FP32, FP16, BF16, FP8, and below.
Quantization
Replacing high-precision weights with lower-precision approximations to save memory and compute. INT4 is common in production.
Embedding
A vector that represents an input in a learned space where geometric distance reflects semantic similarity.
Vector
An ordered list of numbers. In ML, almost always a point in a high-dimensional space representing something.
Dimension
One coordinate axis of a vector space. A 1536-dim embedding has 1536 numbers in it. More dimensions = more room for nuance.
Cosine similarity
A way to compare two vectors by the angle between them. Ranges from -1 to 1. The standard metric for vector search.
Vector search
Finding the K closest vectors to a query vector in a large database. Powered by indexes like HNSW that avoid checking every point.
Latent space
The internal vector space where a model represents its inputs. Often surprisingly meaningful — directions correspond to concepts.
Token
The discrete unit a language model processes. Usually a sub-word, a few characters. "Unbelievable" might be three tokens.
Logit
The raw, unnormalized score the model produces for each possible next token. Negative logits are possible.
Softmax
A function that turns a list of numbers into a probability distribution. Exponentiate, then normalize so they sum to 1.
Temperature
A divisor applied to logits before softmax. Higher temperature = flatter distribution = more random output.
Matrix multiplication
The single dominant operation in a neural network. A layer's output = its weight matrix times its input vector. Roughly 99% of inference compute.
Forward pass
Running an input through every layer of the model to produce an output. What happens every time you ask a model anything.
